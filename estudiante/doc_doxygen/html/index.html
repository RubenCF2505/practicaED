Práctica ED — Juego de Letras (Documentación)

Introducción

Este proyecto implementa un juego de letras (tipo Scrabble) con las siguientes responsabilidades:

  • Gestión de un diccionario de palabras válidas.
  • Definición del conjunto de letras y sus puntuaciones.
  • Bolsa de letras que permite extraer letras aleatoriamente.
  • Motor (Solver) que calcula las mejores soluciones dadas las letras disponibles.
  • Aplicación de consola que orquesta la partida (letras).

Estructura del repositorio

  • Directorio include/: cabeceras públicas de los módulos.
  • Directorio src/: implementación en C++ de los módulos.
  • Directorio data/: ficheros de diccionarios y definición de letras.
  • Directorio doc/: documentación adicional y este mainpage.dox.

Módulos principales

  • Dictionary: lectura y gestión del diccionario de palabras.
    • Archivos: include/dictionary.h, src/dictionary.cpp
  • LettersSet: describe las letras disponibles y la puntuación por letra.
    • Archivos: include/letters_set.h, src/letters_set.cpp
  • LettersBag: bolsa de letras con extracción aleatoria.
    • Archivos: include/letters_bag.h, src/letters_bag.cpp
  • Solver: lógica para validar palabras y buscar soluciones óptimas.
    • Archivos: include/solver.h, src/solver.cpp
  • Aplicación: ejecutable de consola que inicia partidas.
    • Archivo: src/letras.cpp (bin/letras)

Uso rápido

Compilar el proyecto (desde la carpeta estudiante):

make -j4

Ejecutar la aplicación de ejemplo:

./bin/letras ./data/diccionario.txt ./data/letras.txt 8 L

Generar documentación (Doxygen)

Si el Doxyfile ya está en el proyecto puedes ejecutar la tarea correspondiente (Makefile):

make doc_doxygen

Tras generarla, abre doc_doxygen/html/index.html o el html/index.html que cree tu sistema de build.
